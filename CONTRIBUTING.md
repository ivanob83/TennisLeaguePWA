# Contributing

## Commit Messages

Use Conventional Commits:

- feat: new functionality
- fix: bug fix
- docs: documentation only
- chore: tooling, build, or dependencies
- refactor: code change without behavior change
- test: tests only

Examples:
- feat: add match results form
- fix: prevent self-match creation
- docs: update MVP scope

## Formatting and Linting

- Run `npm run lint` and `npm run format` before pushing changes.
- Use `npm run lint:fix` or `npm run format:fix` to auto-fix when possible.
